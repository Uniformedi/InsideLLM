LiteLLM API Endpoints

All endpoints require a Bearer token (Authorization: Bearer <key>).
Access via https://<host>/litellm/ (through Nginx) or http://<host>:4000/ (direct).

OpenAI-Compatible (via /v1/)

MethodEndpointPurpose
GET/v1/modelsList available models
POST/v1/chat/completionsChat completions (primary API for Claude)
POST/v1/completionsText completions

Internal Connection Details

These services are internal to the Docker network. Access them via pgAdmin or SSH into the VM/WSL.

PostgreSQL

Host: postgres (Docker network) Port: 5432 Database: litellm User: litellm Password: <postgres_password> # Convenience views (created by post-deploy): spend_logs, audit_log, users, teams, api_keys, daily_user_spend, daily_team_spend, error_logs, models, budgets

Redis

Host: redis (Docker network) Port: 6379 Max Memory: 256 MB Eviction: allkeys-lru # Stores: Rate limit counters (RPM/TPM per user) Budget enforcement (daily/monthly) LiteLLM operational state

Claude Code CLI Setup

# PowerShell $env:ANTHROPIC_BASE_URL = "http://localhost:4000" $env:ANTHROPIC_AUTH_TOKEN = "<your-litellm-api-key>" # Bash / Linux export ANTHROPIC_BASE_URL="http://localhost:4000" export ANTHROPIC_AUTH_TOKEN="<your-litellm-api-key>"
