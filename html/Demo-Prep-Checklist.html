If things go wrong — troubleshooting tree

Agent doesn't respond in OWUI
Reload the OWUI tab. Verify model is still Dispute Handler. If still stuck, docker logs insidellm-litellm --tail 30; look for auth or model-routing errors.
OPA fails open (doesn't deny when it should)
docker logs insidellm-opa to inspect. Pivot talking point: "This is why defense in depth matters — the LiteLLM guardrail callback also enforces this, independent of OPA availability." Skip to step 4 if step 3 Turn 2 is unrecoverable.
Approval queue is empty when it shouldn't be
Jump straight to step 5 (audit trail). Return to approval concept verbally during the close — don't stall.
Audit endpoint returns 500
Don't live-debug. Narrate the hash-chain concept verbally and hold up the Executive Summary page (it mentions hash-chained audit). Move to step 6.
Policy editor UI won't load .rego
Open the file in a plain browser tab via github.com/uniformedi/insidellm/…/tier_fdcpa_regulated.rego. Backup URL should be bookmarked.
Whole network drops
Pivot to the backup screen recording (if available) or the phone screenshots. Walk through each image with the same talk tracks. Leave-behind Executive Summary is the takeaway.
Someone asks a question you can't answer
"Great question — let me follow up with the specifics after the demo." Note it down visibly. Do not improvise on compliance, legal, or roadmap commitments.
