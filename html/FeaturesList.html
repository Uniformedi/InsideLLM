InsideLLM — Capability Catalog

Every shipping feature of the platform, organized by functional area, with version provenance. Version badge indicates when each capability landed.

Version 3.1.0 Cataloged 2026-04-22 Source README.md + CLAUDE.md

InsideLLM is a self-hosted, on-premises AI gateway that brokers all traffic to Anthropic Claude (and optionally local Ollama models) through enterprise-grade DLP scanning, per-user budgets, SSO, OPA policy enforcement, hash-chained audit, and fleet management. A single terraform apply or PowerShell script deploys the full stack. BSL 1.1 licensed; converts to Apache 2.0 on 2030-04-11.

Governance & Compliance

Humility Alignmentdemo
v3.0
Mandatory AI alignment policy based on the SAIVAS framework from Uniform Gnosis, Vol. I. Enforced at four layers: prompt injection, LiteLLM guardrail, OPA policy, OWUI pipeline. Cannot be disabled.
OPA Policy Enginedemo
v3.0
Open Policy Agent runtime with 6 industry policies: HIPAA, FDCPA, SOX, PCI-DSS, FERPA, GLBA. Obligation execution pipeline (audit.log, filter.fields, require.attestation, review.queue). Fails closed.
Guardrail Profilesdemo
v3.1
Six OPA bundles: tier_unrestricted, tier_general_business, tier_financial_regulated, tier_fdcpa_regulated, tier_hipaa_regulated, tier_custom. 46/46 unit tests pass via opa test.
Hash-Chained Audit Integritydemo
v3.0
SHA-256 chain on every governance event (sync, proposals, approvals, snapshots, agent lifecycle). Verification endpoint. Tamper-evident — cannot edit a row without breaking the chain.
Enterprise Governance Hub
v3.0
FastAPI service for central repository sync (PostgreSQL / MariaDB / MSSQL), change management with supervisor approval, AI-powered governance advisor.
AI Governance Framework Compliance
v3.0
Governance tier classification, data classification, AI Ethics Officer tracking, configurable log retention.
Industry Keyword Templates
v3.0
12 industry presets (Collections, Healthcare, Financial, Legal, Insurance, etc.) with curated regulatory keyword dictionaries.
Keyword Analysis Engine
v3.0
PostgreSQL full-text search on API requests with materialized views, topic distribution, and flagged-request detection.
Agent Manifest Systemdemo
v3.1
Declarative agent manifests (JSON Schema + pydantic). CRUD router at /api/v1/agents/*. Manifest-to-runtime translation under active development (P1.2).

Security & Data Protection

DLP at Gatewaydemo
core
PII / PHI / credential / financial scanning at LiteLLM. Covers all clients (Open WebUI, Claude Code CLI, direct API). Scans inbound user messages and outbound assistant responses. File content scanned via Open WebUI inlined extraction.
Supply Chain Firewall (SCFW)
v2.0
Datadog's supply-chain-firewall wraps pip to block known-malicious PyPI packages before installation.
TLS 1.2 / 1.3 Termination

CVE Scanning (Trivy)
v3.0
Continuous container vulnerability scanning.
Container Patching (Watchtower)
v3.0
Automated container image updates for upstream security patches.
PostgreSQL Backup Cron
v3.0
Scheduled database backups with retention policy.
Secrets Generation
core
Terraform random_password resources generate LITELLM_MASTER_KEY and service credentials at deploy time; injected via cloud-init.

Identity & Access

Unified SSO Across All Services
v3.1
Single IdP app registration (Azure AD or Okta) shared across Open WebUI, Grafana, LiteLLM, and the Admin Command Center. OIDC env vars auto-injected per service.
SSO Group-to-Team Mapping
v3.0
Azure AD / Okta groups auto-map to LiteLLM teams with per-group budgets, rate limits, and model access.
Active Directory Authentication
v3.1
When domain-joined without a cloud IdP, Admin Center uses LDAP bind against AD with group-based access control. Uses ldap3 pure-Python library.
Admin Center Auth — 3 Modes
v3.1
OIDC (cloud SSO), LDAP (on-prem AD), or open (no auth). Auto-detected from deployment config. Nginx auth_request delegates to Governance Hub JWT validation.
Per-User API Keys
core
Individual authentication tokens per user; replaces shared-key anti-pattern.
Per-User / Per-Team Budgets
core
Daily spend caps enforced in real-time via PostgreSQL + Redis counters.
Rate Limiting
core
RPM / TPM enforcement per key, sub-millisecond check via Redis 7.

AI & Model Layer

LiteLLM API Gatewaydemo
core
OpenAI-compatible proxy on port 4000. Authentication, model routing, budgets, rate limiting, audit callbacks. Exposed at /v1/ and /litellm/.

Local LLM Support (Ollama)

Run open-source models locally alongside Claude. Qwen 2.5 Coder 14B + Qwen 2.5 14B default. GPU acceleration via WSL2 or Hyper-V DDA.
DocForge (File Generation)
v3.0
Node.js / Express service generating DOCX, XLSX, PPTX, PDF, CSV, ODF via LibreOffice headless. Exposed as Open WebUI Tool.
AI System Designer
v3.0
Open WebUI Tool that designs deployments, estimates costs, recommends configs, plans multi-instance fleet architectures.
External Data Connectors
v3.0
Query PostgreSQL, MySQL, MSSQL, REST APIs with team-based RBAC, row filtering, field masking, full audit logging.

Deployment & Operations

Terraform + Hyper-V Deployment
core
Single terraform apply deploys full Debian 12 VM on Windows Hyper-V. 8 vCPU / 32 GB RAM default. cloud-init provisioning.
WSL2 Deployment Path
v2.0
Single PowerShell script, no Terraform or Hyper-V. Native GPU passthrough.
Setup Wizard
v2.0
Interactive HTML form at html/Setup.html. Generates terraform.tfvars. Restructured in v3.1 to 9 steps (8 for WSL2) with optional-services step.
PowerShell-Native ISO Creation
v3.0
cloud-init ISO creation without WSL or Windows ADK. Pure .NET via New-CloudInitIso.ps1.
Local Package Cache
v3.1
Primary VM auto-runs apt-cacher-ng + Docker registry pull-through. Peer VMs skip ~1.5 GB apt + ~4 GB Docker image traffic per deploy (~20 min saved).
Claude Code CLI on Every VM
v3.1
Post-deploy auto-installs Claude Code CLI for the admin user. Pre-seeds /opt/InsideLLM/CLAUDE.md with per-VM context so operators get an AI assistant scoped to that box out of the box.
Apache Guacamole (Optional)
v3.1
Browser-based RDP/VNC/SSH gateway at /remote/. Enable via guacamole_enable = true. Seeds Postgres, rotates admin, creates insidellm-admin, pre-populates connections. LDAP auth extension when AD enabled.
Platform Versioning
v3.1
VERSION file at project root wired through Terraform, Docker labels, Governance Hub, Admin UI topbar. Fleet tracks per-node versions with outdated detection.
Monitoring Provisioning Script
v3.1
scripts/provision-monitoring.sh idempotently configures Grafana datasources / dashboards / alerts / Slack contact point, Uptime Kuma monitors + Slack, LiteLLM Slack alerting.
Port Forwarding Script
v2.0
Expose all services to LAN clients via Port-Forward-InsideLLM.ps1.
