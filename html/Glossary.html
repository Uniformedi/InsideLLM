InsideLLM Platform

Governance Hub
The FastAPI service (configs/governance-hub/) that provides the platform's admin API, fleet sync, change-management workflow, and policy-compilation endpoints. Exposed at /governance/api/v1/. One Governance Hub per InsideLLM instance; optionally syncs to a central DB for fleet-wide visibility.
LiteLLM gateway layer
Open-source API gateway at port 4000 providing OpenAI-compatible endpoints across multiple model providers. InsideLLM uses it as the enforcement point for DLP, authentication, budgets, rate limits, and OPA policy invocation via callbacks. Endpoints: /v1/ and /litellm/. Upstream: github.com/BerriAI/litellm
Open WebUI OWUI — chat interface
Browser-based chat UI where users interact with AI. Supports RAG (document upload + Q&A), custom pipelines (DLP, OPA enforcement), and "Custom Models" (published agents appear here).
DocForge
Node.js / Express service that generates DOCX, XLSX, PPTX, PDF, CSV, ODF files using LibreOffice headless. Exposed as an Open WebUI Tool so agents can author documents from chat.
Agent Manifest
YAML file declaring an agent's identity, instructions, guardrail profile, allowed models, daily budget, knowledge scope, and visibility. Schema-validated by pydantic at configs/governance-hub/src/schemas/agents.py. Published via POST to /api/v1/agents/, then transitioned from draft to published state.
Action Catalog
Tenant-scoped YAML catalog of function-calling actions an agent may invoke — e.g., lookup_account, draft_fdcpa_letter, schedule_callback. Uploaded via /api/v1/actions/upload.
Guardrail Profile
Named bundle of OPA policies assigned to an agent. Six profiles ship today: tier_unrestricted, tier_general_business, tier_financial_regulated, tier_fdcpa_regulated, tier_hipaa_regulated, tier_custom. 46/46 unit tests pass per opa test.
Manifest-to-Runtime Translator
The service (under active P1.2 development in configs/governance-hub/src/services/agent_translator.py) that takes a published manifest and provisions the LiteLLM virtual key, OWUI Custom Model registration, and OPA metadata bindings. Idempotent via manifest_hash.
Manifest Hash
SHA-256 of the canonicalized manifest YAML. Used for idempotent provisioning — if unchanged, translator skips; if changed, mints new resources + tears down old.
Runtime Sync State
Field on a published agent indicating whether the translator has successfully provisioned runtime resources. Values: pending, provisioning, provisioned, failed.
Fleet
A collection of InsideLLM instances managed through a central Governance Hub. Each instance has a role (primary, gateway, workstation, voice, edge). Per-node version tracking + outdated detection in the Admin UI.
Portfolio View
Admin-UI dashboard at /governance/portfolio that rolls up every instance in a parent-organization's fleet — instance count, average compliance score, at-risk flags, industry distribution. The "32 companies, one view" frame for the 2026-05-12 demo.
Break-glass admin account
Emergency admin account (insidellm-admin) using LITELLM_MASTER_KEY from /opt/InsideLLM/.env. Accepted by OWUI, Grafana, LiteLLM, Uptime Kuma, pgAdmin, and Guacamole on fresh deploys. Used for recovery when SSO is down.

Governance & Compliance

Humility Alignment
InsideLLM's mandatory AI alignment policy implementing the SAIVAS framework. Four-layer enforcement: prompt injection (soft), LiteLLM guardrail (hard), OPA policy (enterprise overlay), OWUI pipeline (frontend). Cannot be disabled. Canonical source: github.com/uniformedi/humility-guardrail (MIT, pip-installable).
SAIVAS framework — published IP
Sentient AI Value Alignment Standard. Framework described in Uniform Gnosis, Volume I by Dan Medina. The Humility policy implements its alignment principles. Attribution documented in the project NOTICE file.
Hash-Chained Audit
SHA-256 chain on every governance event (agent lifecycle, policy change, approval, action invocation). Each entry references the previous entry's hash; tampering with any entry breaks the chain. Verification endpoint at /governance/api/v1/audit/verify.
Approval Queue
Admin-UI queue at /governance/changes for actions requiring human review before execution — e.g., send_letter. Approver identity + timestamp + diff logged to the hash-chained audit.
Materiality Tier
Three-level classification from the AI Governance Framework. Tier 1: material consumer decisions (full governance). Tier 2: operational AI applications (standard governance). Tier 3: routine productivity tools (lightweight oversight).
Governance Framework
Document at AI_Governance_Framework.md describing the tiered-governance approach, six pillars of ethical AI (Fairness, Transparency, Privacy, Accountability, Safety, Societal Impact), and regulatory mapping. Inspired by Cassie Kozyrkov (Decision Intelligence) and Mo Gawdat (ethics).
Obligation OPA output
Instruction returned by an OPA policy directing InsideLLM to perform a post-decision action. Four defined types: audit.log, filter.fields, require.attestation, review.queue. OPA is pure — it returns obligations; InsideLLM enforces.
Defense in Depth
Enforcement principle: the same policy is enforced at multiple independent layers so a failure at one layer does not produce a governance escape. Humility is enforced at four layers explicitly for this reason.
Fail Closed
Security principle: when a control fails, the default action is denial, not permission. OPA errors → denial. DLP service unreachable → request blocked (configurable).

AI / LLM

LLM Large Language Model
Family of AI models trained on large text corpora (Claude, GPT, Gemini, Qwen, Llama, etc.). InsideLLM routes traffic to whichever LLM(s) the agent manifest declares as allowed_models.
Ollama
Local LLM inference runtime. Runs open-source models (Qwen 2.5 Coder 14B + Qwen 2.5 14B are defaults) on-premises with GPU acceleration. Zero token cost, full data residency.
RAG Retrieval-Augmented Generation
Technique where the model's response is grounded in documents retrieved from a vector store rather than its training data alone. Open WebUI's RAG uses local embeddings — documents never leave the network.
DLP Data Loss Prevention
Scanning of user messages and assistant responses for patterns indicating leakage of PII, PHI, credentials, financial data, or custom-regex patterns. Enforced at configs/litellm/callbacks/dlp_guardrail.py. Configured via dlp_* Terraform variables.
Prompt Injection
Attack where user input attempts to override or manipulate the system prompt or tools the agent has access to. InsideLLM's guardrails evaluate at the LiteLLM layer regardless of what the prompt says.
Langfuse
LLM observability platform (MIT-licensed, self-hostable). InsideLLM registers Langfuse as a LiteLLM callback so every API call is logged with prompt, completion, cost, and latency.
RPM / TPM
Requests-Per-Minute / Tokens-Per-Minute. Rate limits enforced per LiteLLM key, backed by Redis counters.
Streaming
Response delivered as chunks rather than waiting for the full completion. InsideLLM supports streaming; DLP outlet scanning can redact mid-stream on chunk boundaries.

Policy Engine (OPA / Rego)

OPA Open Policy Agent
CNCF-graduated open-source policy engine. Receives JSON input, evaluates Rego rules, returns {allow, deny_reasons, obligations}. InsideLLM runs OPA as a sidecar container at port 8181. Never calls external services — enforcement is InsideLLM's job after OPA returns.
Rego
Declarative policy language used by OPA. Policy files live at configs/opa/policies/*.rego. Readable by non-developers (including General Counsel) and testable via opa test.
Policy Bundle
A set of Rego files shipped together as a guardrail profile. Six bundles in configs/opa/policies/profiles/: tier_unrestricted, tier_general_business, tier_financial_regulated, tier_fdcpa_regulated, tier_hipaa_regulated, tier_custom.
OPA Input Schema
JSON structure passed to OPA per request: user, agent_meta, request, content. Constructed by humility_guardrail.py::_build_opa_input. Schema extended in P0.2.
agent_meta
Section of the OPA input containing agent-scoped metadata: guardrail_profile, data_classes_in_context, allowed_models, baa_models, max_actions_per_session, token_budget_per_session.
Policy-as-Code
Compliance principle: policies are machine-readable source files under version control, not prose documents. They are inspectable, testable, diffable, and enforced by a runtime. Contrast: policy-as-documents (PDFs that are aspirational).

Deployment & Infrastructure

Hyper-V
Microsoft's Windows-native hypervisor. InsideLLM's primary deployment target — a Terraform provider creates VMs on a Windows 11 Pro or Server host. Zero cost; IT departments already have expertise.
WSL2 Windows Subsystem for Linux
Lightweight Linux runtime on Windows. InsideLLM's alternative deployment path: single PowerShell script, no Terraform required, native GPU passthrough. Ideal for development + evaluation.
Terraform
HashiCorp's infrastructure-as-code tool. terraform apply renders the compose file, nginx config, secrets, and cloud-init ISO, then creates the Hyper-V VM and boots it. Source under HashiCorp's BSL 1.1 license.
cloud-init
Linux VM first-boot automation framework. InsideLLM's Terraform path builds a cloud-init ISO (PowerShell-native in v3.0, no WSL required) that runs post-deploy scripts on first VM boot.
tfvars terraform.tfvars
Variable values file for Terraform. html/Setup.html wizard generates one. html/DefaultDeployment.html documents all 129 defaults.
post-deploy.sh.tpl
Terraform template for the VM's post-first-boot provisioning script. Installs Docker Compose, starts services, seeds the admin account, and in v3.1 installs Claude Code CLI with a pre-seeded CLAUDE.md.
SCFW Supply Chain Firewall
Datadog's open-source pip wrapper (github.com/DataDog/supply-chain-firewall). Blocks known-malicious PyPI packages before installation. Active on the WSL2 path; integrated into the Docker build for package installs.
Watchtower
Container auto-update tool. Monitors container images for upstream updates and applies them to the running stack. Configured with Slack alerting for update events.
Trivy
Open-source container CVE scanner (Aqua Security). Runs continuously against the InsideLLM container images, reports vulnerabilities to Grafana dashboards.
Local Package Cache
v3.1 feature. Primary VM runs apt-cacher-ng + Docker registry pull-through cache. Peer VMs in the fleet point at the primary to skip ~1.5 GB apt + ~4 GB Docker traffic per deploy (~20 min saved).
Apache Guacamole
Clientless remote-access gateway. v3.1 ships an optional deployment (guacamole_enable = true) providing browser-based RDP/VNC/SSH at /remote/. LDAP auth extension installed when AD enabled.

Security

OIDC OpenID Connect
Modern SSO protocol built on OAuth 2.0. InsideLLM supports Azure AD and Okta as OIDC providers, with group-to-team mapping for LiteLLM teams.
LDAP
Lightweight Directory Access Protocol — the on-prem authentication option. When a Hyper-V VM is domain-joined to Active Directory, InsideLLM uses LDAP bind with group-based access control via the ldap3 Python library.
BAA Business Associate Agreement
HIPAA agreement between a covered entity and a third party that handles PHI. For InsideLLM deployments using cloud models for PHI workflows, the customer holds the BAA directly with Anthropic or OpenAI; InsideLLM enforces the routing via baa_models manifest field.
nginx auth_request
Nginx module that delegates authentication to an upstream service. InsideLLM's Admin Center Auth (three modes: OIDC / LDAP / Open) uses auth_request to delegate JWT validation to the Governance Hub.
JWT JSON Web Token
Token format used by InsideLLM for admin and API authentication. Minted by /governance/auth/token; validated by Governance Hub on incoming requests.

Regulations

FDCPA 15 U.S.C. § 1692
Fair Debt Collection Practices Act — federal law restricting time/place/frequency of consumer-debt communications. Enforced by the CFPB and state AGs. Policy bundle: tier_fdcpa_regulated.
HIPAA 45 CFR 160/162/164
Health Insurance Portability and Accountability Act. Protects PHI. Enforced by HHS OCR. Policy bundle: tier_hipaa_regulated.
GLBA 15 U.S.C. § 6801
Gramm-Leach-Bliley Act. Financial-institution customer-information privacy. Three parts: Safeguards Rule, Privacy Rule, Pretexting Rule. Enforced by the FTC and federal banking regulators.
FCRA 15 U.S.C. § 1681
Fair Credit Reporting Act. Regulates consumer credit reporting accuracy and dispute rights. Enforced by CFPB + FTC.
ECOA 15 U.S.C. § 1691, Reg B
Equal Credit Opportunity Act. Prohibits credit discrimination on protected-class bases. Implemented by Regulation B (12 CFR 1002).
TCPA 47 U.S.C. § 227
Telephone Consumer Protection Act. Restricts calls to mobile numbers, autodialers, pre-recorded messages. Enforced by the FCC and FTC.
CFPB Regulation F 12 CFR Part 1006
CFPB rule implementing FDCPA with specific limits (7-in-7 call frequency rule, email/text consent, validation-notice content).
FTC Act § 5 15 U.S.C. § 45
Prohibits unfair or deceptive acts or practices. Applied by the FTC to AI-generated deceptive claims and undisclosed automation.
SOX Sarbanes-Oxley Act
Sarbanes-Oxley Act. Internal controls over financial reporting; §302 management certification; §404 auditor attestation. Policy bundle: tier_financial_regulated.
PCI-DSS
Payment Card Industry Data Security Standard. Contractually binding on merchants / processors. DLP detects + blocks PAN and CVV patterns.
FERPA 20 U.S.C. § 1232g
Family Educational Rights and Privacy Act. Student educational records — disclosure restrictions, consent requirements. Enforced by the US Department of Education.
NIST AI RMF
Risk Management Framework for AI from the US National Institute of Standards and Technology. Govern / Map / Measure / Manage. Voluntary. InsideLLM aligns at the control level.
ISO/IEC 42001:2023
International management-system standard for AI. Certifiable by accredited auditors. InsideLLM aligns at the control level; formal certification targeted for 2027.
EU AI Act
EU Regulation 2024/1689. Phased enforcement 2025-2027. High-risk AI systems subject to conformity assessment, transparency, human oversight obligations. InsideLLM's Conformity Profile roadmapped for v3.3.
Colorado AI Act SB24-205
Effective 2026-02-01. Covers consequential AI decisions in employment, education, financial services, healthcare, housing, insurance, legal services, essential government services.

Misc / Uniformedi-specific

Uniformedi LLC
The legal entity behind InsideLLM and Secular. Authored by Dan Medina. BSL 1.1 licensor of InsideLLM; Apache 2.0 licensor of Secular.
BSL 1.1 Business Source License
Source-available license permitting all use except offering the software as a competing commercial service to third parties. Converts automatically to Apache 2.0 on a defined date — for InsideLLM, 2030-04-11.
Secular
Sibling FOSS project from Uniformedi — Apache 2.0 — an AGENTS.md maintenance toolchain for FOSS AI coding agents. Pre-MVP, design documented. Sibling to InsideLLM; not a dependency.
AAIF Agentic AI Foundation
Linux Foundation project formed 2025-12-09. Three founding projects: Anthropic's MCP, Block's goose, OpenAI's AGENTS.md. Provides neutral governance for agentic AI standards.
AGENTS.md
AAIF-stewarded open standard — a Markdown file at the root (or nested) of a repository giving AI coding agents project-specific guidance. ~60,000 repos adopted. Supported by Cursor, Copilot, Codex, Amp, Jules, Factory, and more.
CLAUDE.md
Anthropic's per-repo guidance convention for Claude Code. Coexists with AGENTS.md; Claude Code reads nested CLAUDE.md files automatically.
Nested Dan informal term
Internal term for the per-VM Claude Code CLI pre-seeded with scoped context so an operator SSH'd into any InsideLLM VM has an AI assistant tailored to that box. v3.1 feature. See docs/ClaudeCode-On-VMs.md.
humility-guardrail
Standalone Python package (MIT, pip-installable) at github.com/uniformedi/humility-guardrail. Canonical source for the Humility alignment rules, prompts, and base OPA policy. InsideLLM wraps it with Redis prompt loading, OPA delegation, and governance-hub audit logging.
Parent Organization
Internal reference name for the 2026-05-12 demo customer — a portfolio-owner of 32 companies across varied industries. Target: convert to design-partner + first fleet deployment.
Dispute Handler
The showcase declarative agent for the Parent Organization demo. Handles consumer debt-dispute workflows under tier_fdcpa_regulated. Actions: lookup_account, draft_fdcpa_letter, schedule_callback, approval-gated send_letter.
